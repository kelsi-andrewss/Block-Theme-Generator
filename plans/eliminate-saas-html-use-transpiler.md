# Eliminate SAAS_FRONT_PAGE_HTML — use transpiler as single bridge

Story: (pending)
Agent: quick-fixer

## Context

The gallery SaaS theme uses a hardcoded `SAAS_FRONT_PAGE_HTML` WP block string that drifts from the React design reference at `/templates/saas/page.tsx`. Replace it with canonical inline-styles JSX that runs through `transpileJSXToBlocks()` so the transpiler is the single source of WP markup.

## What changes

| File | Change |
|---|---|
| `src/lib/generators/saas-front-page-jsx.ts` | **NEW** — Export `generateSaasFrontPageJsx(): string` that returns the full SaaS front page as inline-styles JSX (hero, logos, features, testimonials, pricing, CTA). Features section interpolates `SAAS_FEATURES` data. This is the equivalent of `/templates/saas/page.tsx` but with inline styles instead of Tailwind classes, and using the WP component conventions from `block-markup.ts` (TemplatePart, Buttons, Button, etc). |
| `src/lib/generators/saas-template.ts` | Remove the 400-line `SAAS_FRONT_PAGE_HTML` template literal. Replace with: `import { transpileJSXToBlocks } from '../transpiler/jsx-to-blocks'; import { generateSaasFrontPageJsx } from './saas-front-page-jsx'; export const SAAS_FRONT_PAGE_HTML = transpileJSXToBlocks(generateSaasFrontPageJsx());`. Keep `COLOR_MAP` and `featuresGrid` logic — move them into the new JSX file. |
| `src/app/app/page.tsx` | No changes needed — `import { SAAS_FRONT_PAGE_HTML }` stays the same since the export name is unchanged. |

## Tasks

1. Read `/templates/saas/page.tsx` fully to understand all 6 sections (hero, logos, features, testimonials, pricing, CTA)
2. Read `src/lib/generators/saas-template.ts` fully to understand the WP-specific values (color-mix, preset vars, SVG icons, SAAS_FEATURES interpolation)
3. Create `src/lib/generators/saas-front-page-jsx.ts`:
   - Import `SAAS_FEATURES` from `../data/saas-features`
   - Export `generateSaasFrontPageJsx()` that returns a JSX string using inline `style={{}}` objects (NOT Tailwind classes)
   - Use WP preset color vars (`var(--wp--preset--color--primary)`, etc.) and `color-mix()` for colors — match the existing `saas-template.ts` values, NOT the Tailwind vars from `page.tsx`
   - For dynamic features: build feature JSX from `SAAS_FEATURES` array and interpolate into the template string
   - Include SVG icons inline (copy from existing saas-template.ts)
   - The JSX must use standard HTML elements (`section`, `div`, `h2`, `p`, `a`, `ul`, `li`, `span`) with inline style objects
   - For buttons: use `<a role="button" href="..." style={{...}}>` so the transpiler maps to `wp:button`
   - For template parts: use `<TemplatePart slug="header" tagName="header" />` and `<TemplatePart slug="footer" tagName="footer" />`
4. Modify `src/lib/generators/saas-template.ts`:
   - Remove the `SAAS_FRONT_PAGE_HTML` template literal and the `featuresGrid`/`COLOR_MAP` code that was only used by it
   - Add imports for `transpileJSXToBlocks` and `generateSaasFrontPageJsx`
   - Export `SAAS_FRONT_PAGE_HTML = transpileJSXToBlocks(generateSaasFrontPageJsx())`
5. Run `npx tsc --noEmit` to verify types
6. Run a quick sanity check: `npx tsx -e "import { SAAS_FRONT_PAGE_HTML } from './src/lib/generators/saas-template'; console.log(SAAS_FRONT_PAGE_HTML.substring(0, 500));"` to verify the transpiler produces valid WP block output

## Acceptance criteria

- `SAAS_FRONT_PAGE_HTML` is generated by running JSX through `transpileJSXToBlocks()`, not hardcoded
- The exported string starts with `<!-- wp:template-part` and contains valid WP block markup
- TypeScript compiles without errors
- The `saas-template.ts` no longer contains raw WP block markup — only the transpiler call
- `SAAS_FEATURES` data is still interpolated into the features section
